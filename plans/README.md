## Direction findings (options for the maintainer, not bugs)

Recorded from the audit's direction category; these are product choices, not defects:

- **Point-in-time index constituents.** The backtests use *current* constituents backtested
  historically (survivorship bias, already documented as a known ~+194%→~+145% haircut). Building a
  true point-in-time universe from full-mainboard data would de-bias results but needs historical
  constituent/delisting data the free sources don't fully provide. A spike, not a quick fix.
- **Consolidate to one strategy.** The repo now carries multiple strategy lineages (golden-cross
  research, CSI1000 factor pipeline, CS core-satellite). Picking one canonical live path and
  archiving the rest would cut the surface area that findings D/E/F all touch.
- **`AI_CHAIN` staleness.** The AI-satellite theme→ticker map in `portfolio.py` is hardcoded and
  will drift as the market changes; it needs a refresh policy or a data-driven source. This is a
  maintenance-process decision for the user (it is an explicit "信仰仓" bet, not alpha).
