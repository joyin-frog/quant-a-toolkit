# UI 优化 Spec（quant-a web，2026-06-17）

桌面 web（Next.js 16 + shadcn/ui + Tailwind v4，**不需要移动端**）。两页：
`web/app/page.tsx`（月度调仓：左参数卡 + 右指标/净值图/下单清单）、
`web/app/portfolio/page.tsx`（实盘记账：持仓盈亏 / 实盘绩效 / 执行评分卡 / 收益归因 / 因子体检 / 成交流水）。

约束（贯穿全部 10 项）：
- **A 股红涨绿跌**配色语义不变：`text-gain`(红/涨)、`text-loss`(绿/跌)、`text-primary`(蓝)。已有的 `pct()/yuan()/tone()` 辅助函数继续用。
- 用 **shadcn 组件**,不要手搓样式 div 替代已有组件;颜色用语义 token(`bg-muted`/`text-muted-foreground`/`bg-card`…),不要写死 hex。
- 数字列用 `tabular-nums`。图标用 lucide(已用),不要 emoji。
- 每做完一项跑 `npx tsc --noEmit` 保持通过;全部做完跑 `npx next build` 必须成功(允许保留既有的 "inferred workspace root" 与 "NFT list" 两条 benign 构建告警,**不得新增其它告警/报错**)。
- 不改后端 Python、不改 API 返回结构(纯前端)。需要的 shadcn 组件用 `npx shadcn@latest add <name>` 装;需要的 npm 包用 `npm install`。

## 1. 下单清单分腿可视化（主页）
- "共 N 只" → **"共 N 只（核心 X · AI 卫星 Y）"**(按 `holdings_list` 的 `sleeve` 统计)。
- 清单表格下方加一行**分腿小计**:`核心投入 ¥X · AI 卫星投入 ¥Y · 现金 ¥Z`(从 `holdings_list` 的 `sleeve`+`cost` 汇总,现金用已有 `cash_left`)。
- **验收**:主页生成清单后,标题/小计能看到核心与 AI 的只数和金额拆分。

## 2. 慢操作骨架屏（主页生成清单 / 实盘页生成绩效·生成复盘）
- 这三个操作 30-60s。loading 期间用 **shadcn `Skeleton`**(已装)画出结果区骨架(指标卡若干 + 图表占位条),替代"右侧空白只转圈"。
- **验收**:点生成后、数据回来前,右侧显示骨架占位;数据到后正常渲染。

## 3. 术语 ⓘ 解释气泡
- 装 `npx shadcn@latest add tooltip`(注意 RSC:页面是 `"use client"`)。
- 给这些 jargon 指标的标签加一个小 `ⓘ`(lucide `InfoIcon`,`size`走组件默认)+ Tooltip 一句话解释:
  - 主页:夏普、最大回撤、年化、滚动12月。
  - 实盘页:**跟踪误差**(实盘与策略每日偏离幅度,越小越贴)、**对回测损耗**(实盘比同期策略回测差多少=执行滑点/纪律代价)、**对基准超额**、夏普、**IC_IR/rank-IC**(因子排名与未来收益的相关稳定度)、**衰减**。
- **验收**:这些标签旁有可悬停的 ⓘ,内容为上面的人话解释。

## 4. 净值图：坐标轴 + 空态 + 加载态
- Y 轴加标签"净值(基1)";已有的多线(实盘实线/回测虚线/基准)保持。
- 图表在**未生成**时显示空状态("填参数 → 生成后这里出净值曲线"),**加载中**显示 Skeleton 占位(配合第 2 项)。
- **验收**:无数据/加载/有数据三态都不是空白坐标框。

## 5. 主页下单清单表格卡内滚动
- 把主页下单清单 `Table` 包进 `max-h-[26rem] overflow-y-auto`,表头 `TableHeader` 加 `bg-card sticky top-0 z-10`(与 `/portfolio` 已有表格一致)。
- **验收**:清单超过约 12 行时卡内滚动、表头吸顶,不再把整页拖很长。

## 6. 密集数据表可排序
- 给这几张表的列头做**点击排序**(useState 存 sortKey+方向,排序副本数据;给 `TableHead` 加 `aria-sort` 和指示箭头 lucide `ChevronUpIcon`/`ChevronDownIcon`,加 `cursor-pointer`):
  - 收益归因贡献榜(按 贡献/自身/盈亏)、当前持仓盈亏(按 浮盈亏/今日)、因子体检(按 近一年IC/衰减)。
- **验收**:点这些表的数值列头能升/降序切换,当前排序列有箭头 + `aria-sort`。

## 7. 暗色模式切换 + 语义 token
- `npm install next-themes`;在 `web/app/layout.tsx` 包 `ThemeProvider`(`attribute="class"`,`defaultTheme="system"`,`enableSystem`);加一个**主题切换按钮**(lucide `SunIcon`/`MoonIcon`,放两页 header 右侧,`"use client"`)。`html` 已有/需要 `suppressHydrationWarning`。
- 顺手确认两页用的是语义 token(没有写死 hex);如发现写死的颜色,改成 token。
- **验收**:切换按钮能在 亮/暗/跟随系统 间切,刷新后保持;暗色下文字不糊(主文≥4.5:1、次文≥3:1,目测明显可读)。

## 8. 主页首屏空状态
- 主页右侧在生成前用 **shadcn `Empty`**(`npx shadcn@latest add empty`,若 registry 没有就用 Card+居中文案等价实现)给"① 刷新行情 → ② 生成清单"两步引导,替代空白。
- **验收**:未生成时右侧是有引导的空状态而非全白。

## 9. 进度条 + 图表可访问性
- 主页刷新进度条容器加 `role="progressbar"` `aria-valuenow={done}` `aria-valuemin={0}` `aria-valuemax={total}` `aria-label="刷新行情进度"`。
- 两个净值图的容器加 `aria-label` 概括(如"实盘 vs 回测 vs 基准 净值曲线")。
- **验收**:进度条有 progressbar 语义;图表容器有 aria-label。

## 10. 股票代码列等宽（可选锦上添花）
- 下单清单 / 持仓 / 流水 / 归因表里的**股票代码**单元格加 `font-mono`(数字已 tabular-nums)。
- **验收**:代码列等宽对齐。
